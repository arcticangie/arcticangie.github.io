09. Port

Port는 하나의 IP 주소 안에서 어떤 서비스와 통신할 것인지를 구분하는 번호이다. IP 주소가 목적지를 찾는 역할이라면, Port는 목적지 안에서 어떤 프로그램과 연결할지를 결정한다. 같은 서버에서도 여러 서비스를 동시에 운영할 수 있는 이유가 Port 덕분이다.

왜 중요한가?

  • 서비스별 통신을 구분할 수 있다.
  • 방화벽 정책은 IP와 Port를 기준으로 작성하는 경우가 많다.
  • 네트워크 장애 분석 시 어떤 Port가 열려 있는지 확인한다.
  • 침해사고 분석에서도 공격 대상 서비스를 식별하는 기준이 된다.

핵심 이론

구분범위설명
Well-known Port0~1023대표 서비스가 사용하는 포트
Registered Port1024~49151응용 프로그램에서 주로 사용
Dynamic / Ephemeral Port49152~65535클라이언트가 임시로 사용하는 포트

Port프로토콜용도
20/21FTP파일 전송
22SSH원격 접속
23Telnet원격 접속(암호화 미지원)
25SMTP메일 전송
53DNS도메인 조회
80HTTP웹 서비스
110POP3메일 수신

Port는 TCP와 UDP 각각 별도로 관리된다. 예를 들어 TCP 53과 UDP 53은 같은 번호이지만 서로 다른 통신으로 취급한다.

흐름도

Client ↓ 203.0.113.10:443 ↓ HTTPS 서비스 ↓ Web Server

Browser ↓ TCP 연결 ↓ Destination Port : 443 ↓ HTTPS 통신 시작

예시

192.168.0.10 ↓ https://example.com ↓ TCP 443 연결 ↓ 웹 페이지 응답

실무에서 사용하는 경우

  • 웹 서비스 접속이 되지 않으면 TCP 80 또는 TCP 443이 열려 있는지 확인한다.
  • netstat, ss, lsof 명령으로 사용 중인 Port를 확인한다.
  • 방화벽 정책에서 필요한 Port만 허용하고 나머지는 차단한다.
  • 포트 스캔 결과를 통해 어떤 서비스가 실행 중인지 확인한다.

알아두면 좋은 내용

  • IP는 장비를 찾고, Port는 서비스를 찾는다.
  • HTTPS는 기본적으로 TCP 443을 사용한다.
  • Telnet(23)은 암호화되지 않아 SSH(22)를 사용하는 경우가 많다.
  • 클라이언트는 일반적으로 임시(Ephemeral) Port를 사용해 서버와 연결한다.

핵심 요약

  • Port는 하나의 IP 안에서 서비스를 구분하는 번호이다.
  • Well-known, Registered, Dynamic Port로 구분된다.
  • 80(HTTP), 443(HTTPS), 22(SSH), 53(DNS)는 반드시 기억해두는 것이 좋다.
  • 방화벽 정책과 네트워크 분석에서 Port 정보는 가장 중요한 요소 중 하나이다.
  • IP와 Port를 함께 이해해야 통신 흐름을 정확하게 파악할 수 있다.
