11. Router

Router는 서로 다른 네트워크를 연결하는 L3(Network Layer) 장비이다. 목적지 IP 주소를 확인한 뒤 Routing Table을 참고하여 가장 적절한 경로로 패킷을 전달한다. 인터넷 통신은 대부분 Router를 거쳐 이루어진다.

왜 중요한가?

  • 서로 다른 네트워크 간 통신을 가능하게 한다.
  • Routing 동작 원리를 이해할 수 있다.
  • Gateway와 인터넷 연결 구조를 이해하는 기본 장비이다.
  • 기업 네트워크와 클라우드 환경에서 필수적으로 사용된다.

핵심 이론

용어설명
Routing목적지까지의 최적 경로를 선택하는 과정
Routing Table목적지와 전달 경로를 저장하는 테이블
Next Hop다음으로 패킷을 전달할 장비
Default Route일치하는 경로가 없을 때 사용하는 기본 경로
Gateway다른 네트워크로 나가기 위한 출입구

SwitchRouter
L2 장비L3 장비
MAC 주소 기반IP 주소 기반
같은 네트워크 연결다른 네트워크 연결

Router는 패킷을 수신하면 목적지 IP를 확인하고 Routing Table과 비교하여 적절한 인터페이스로 전달한다.

흐름도

PC ↓ Default Gateway ↓ Router ↓ Routing Table 조회 ↓ Internet

목적지 IP 확인 ↓ Routing Table 검색 ↓ Next Hop 결정 ↓ 패킷 전달

예시

192.168.10.20 ↓ 8.8.8.8 접속 ↓ Router가 목적지 경로 선택 ↓ Internet으로 전달

실무에서 사용하는 경우

  • 다른 네트워크와 통신이 되지 않으면 Routing Table을 확인한다.
  • Default Gateway 설정 오류 여부를 점검한다.
  • Traceroute로 패킷 이동 경로를 확인한다.
  • Static Route 또는 Dynamic Route 설정을 관리한다.

알아두면 좋은 내용

  • Router는 OSI 3계층 장비이다.
  • IP 주소를 기준으로 패킷을 전달한다.
  • 기업 환경에서는 Dynamic Routing을 사용하는 경우가 많다.
  • 라우터는 NAT, ACL 등의 기능도 함께 제공하는 경우가 많다.

핵심 요약

  • Router는 서로 다른 네트워크를 연결하는 장비이다.
  • Routing Table을 이용해 최적의 경로를 선택한다.
  • Gateway와 Routing 개념을 함께 이해하는 것이 중요하다.
  • IP 기반 통신의 핵심 장비이다.
  • 기업 인프라에서 반드시 사용되는 네트워크 장비이다.
