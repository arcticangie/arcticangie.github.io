02. TCP/IP 모델

TCP/IP 모델은 현재 인터넷에서 실제로 사용되는 네트워크 통신 모델이다. OSI 7계층이 개념을 설명하기 위한 참조 모델이라면, TCP/IP는 실제 통신이 이루어지는 구조라고 보면 된다. 대부분의 네트워크 장비와 운영체제도 TCP/IP 모델을 기반으로 동작한다.

왜 중요한가?

  • 인터넷 통신의 기본 구조를 이해할 수 있다.
  • 프로토콜이 어느 계층에서 동작하는지 구분하기 쉽다.
  • 패킷 분석과 네트워크 장애 분석의 기준이 된다.
  • 보안 장비의 로그와 정책도 대부분 TCP/IP 구조를 기준으로 해석한다.

핵심 이론

TCP/IP 모델은 4개의 계층으로 구성된다. OSI 모델보다 단순하며 실제 네트워크 환경에서 가장 많이 사용하는 모델이다.

TCP/IP 계층역할대표 프로토콜
Application사용자 서비스 제공HTTP, HTTPS, DNS, SMTP, FTP
Transport종단 간 데이터 전송TCP, UDP
InternetIP 주소 기반 라우팅IP, ICMP, ARP
Network Access실제 네트워크 전송Ethernet, Wi-Fi

OSITCP/IP
Application + Presentation + SessionApplication
TransportTransport
NetworkInternet
Data Link + PhysicalNetwork Access

TCP/IP는 OSI보다 계층 수는 적지만 실제 통신에 필요한 기능은 모두 포함하고 있다. 따라서 실무에서는 OSI와 TCP/IP를 함께 이해하는 것이 중요하다.

흐름도

웹 브라우저 ↓ Application (HTTP) ↓ Transport (TCP) ↓ Internet (IP) ↓ Network Access (Ethernet) ↓ 네트워크 전송

수신 장비 ↓ Network Access ↓ Internet ↓ Transport ↓ Application

예시

https://example.com 접속 HTTP 요청 생성 ↓ TCP 연결 ↓ IP 주소로 라우팅 ↓ Ethernet Frame 생성 ↓ 서버 전달

실무에서 사용하는 경우

  • Wireshark에서 HTTP, TCP, IP 헤더를 계층별로 분석한다.
  • TCP 연결 실패인지, IP 라우팅 문제인지 계층을 기준으로 원인을 구분한다.
  • 방화벽은 IP와 Port 정보를 이용해 통신을 허용하거나 차단한다.
  • DNS, HTTP, HTTPS 같은 서비스가 어느 계층에서 동작하는지 이해하면 로그 분석이 쉬워진다.

알아두면 좋은 내용

  • TCP/IP는 인터넷 표준 프로토콜 집합이다.
  • TCP는 신뢰성 있는 통신, UDP는 빠른 통신을 위해 사용된다.
  • IP는 데이터를 목적지까지 전달하지만 전송 성공 여부는 보장하지 않는다.
  • OSI는 학습용 모델, TCP/IP는 실제 구현 모델이라는 점을 함께 기억하면 좋다.

핵심 요약

  • TCP/IP 모델은 실제 인터넷에서 사용하는 네트워크 모델이다.
  • 총 4개의 계층으로 구성되며 각 계층은 역할이 명확하게 구분된다.
  • HTTP, TCP, IP, Ethernet이 대표적인 프로토콜이다.
  • 실무에서는 OSI보다 TCP/IP 모델을 기준으로 통신을 이해하는 경우가 많다.
  • 네트워크와 인프라 보안을 공부할 때 반드시 이해해야 하는 기본 개념이다.
