08. TCP (Transmission Control Protocol)

TCP는 신뢰성 있는 데이터 전송을 제공하는 전송 계층(Transport Layer) 프로토콜이다. 데이터가 순서대로 전달되는지 확인하고, 손실된 데이터는 다시 전송하기 때문에 웹 서비스, SSH, 데이터베이스처럼 정확한 통신이 필요한 서비스에서 주로 사용된다.

왜 중요한가?

  • 웹 서비스와 대부분의 기업 시스템이 TCP를 사용한다.
  • 패킷 손실 시 재전송을 수행하여 데이터의 신뢰성을 보장한다.
  • Wireshark 분석에서 가장 자주 확인하는 프로토콜이다.
  • 네트워크 장애 분석 시 TCP 연결 상태를 먼저 확인하는 경우가 많다.

TCP는 통신을 시작하기 전에 3-Way Handshake를 수행하고, 연결을 종료할 때는 4-Way Handshake를 수행한다. 또한 Flow Control과 재전송 기능을 통해 안정적인 데이터 전송을 보장한다.

흐름도

3-Way Handshake Client ↓ SYN Server ↓ SYN + ACK Client ↓ ACK 연결 완료

4-Way Handshake FIN ↓ ACK ↓ FIN ↓ ACK 연결 종료

예시

브라우저에서 HTTPS 접속 TCP 연결 생성 ↓ TLS Handshake ↓ HTTP Request ↓ HTTP Response ↓ 연결 종료

실무에서 사용하는 경우

  • 웹 서버 접속이 되지 않으면 먼저 TCP Handshake가 정상적으로 이루어지는지 확인한다.
  • Wireshark에서 SYN, SYN/ACK, ACK 패킷을 확인하여 연결 실패 원인을 분석한다.
  • 방화벽이 TCP 443 또는 TCP 22 포트를 차단하고 있는지 확인한다.
  • 재전송(Retransmission)이 반복되면 네트워크 품질이나 장비 문제를 의심한다.

알아두면 좋은 내용

  • TCP는 데이터의 순서를 보장한다.
  • ACK를 받지 못하면 동일한 데이터를 다시 전송한다.
  • Window Size를 이용해 송수신 속도를 조절한다.
  • HTTPS, SSH, RDP, SMB 등 대부분의 업무용 서비스는 TCP를 사용한다.

핵심 요약

  • TCP는 신뢰성 있는 연결형 전송 프로토콜이다.
  • 3-Way Handshake로 연결을 생성하고 4-Way Handshake로 종료한다.
  • Sequence Number와 ACK를 이용해 데이터 순서와 수신 여부를 관리한다.
  • 재전송과 Flow Control을 지원하여 안정적인 통신을 제공한다.
  • 기업 네트워크와 웹 서비스에서 가장 많이 사용하는 전송 프로토콜이다.
