15. TLS & SSL

TLS(Transport Layer Security)는 네트워크 통신을 암호화하기 위한 보안 프로토콜이다. SSL은 TLS의 이전 버전으로 현재는 대부분 TLS를 사용하지만, 관례적으로 SSL이라는 이름도 함께 사용한다. HTTPS 통신도 TLS를 이용해 데이터를 안전하게 보호한다.

왜 중요한가?

  • 웹 통신에서 개인정보와 인증 정보를 안전하게 보호한다.
  • HTTPS 동작 원리를 이해하는 핵심 기술이다.
  • 인증서를 이용해 서버의 신뢰성을 확인할 수 있다.
  • 보안 장비와 웹 서버를 운영할 때 자주 다루는 기술이다.

핵심 이론

용어설명
TLS현재 사용되는 암호화 통신 프로토콜
SSLTLS 이전 버전(현재는 사용 권장하지 않음)
Certificate서버의 신원을 증명하는 디지털 인증서
CA인증서를 발급하는 기관
Handshake암호화 통신을 시작하기 위한 초기 과정

암호화 방식용도
비대칭키 암호화키 교환 및 인증
대칭키 암호화실제 데이터 암호화

TLS는 Handshake 과정에서 인증서를 검증하고 안전하게 대칭키를 공유한 뒤, 이후부터는 대칭키를 이용해 데이터를 암호화하여 전송한다.

흐름도

ClientHello ↓ ServerHello ↓ Certificate 전달 ↓ Key Exchange ↓ 암호화 통신 시작

브라우저 ↓ HTTPS 요청 ↓ TLS Handshake ↓ 암호화된 데이터 송수신

예시

https://example.com 접속 ↓ 인증서 검증 ↓ TLS 연결 완료 ↓ 암호화된 HTTP 통신

실무에서 사용하는 경우

  • 웹 서버 인증서 만료 여부를 확인한다.
  • HTTPS 접속 오류가 발생하면 TLS 버전과 인증서를 점검한다.
  • 보안 장비에서 TLS Inspection 기능을 사용하는 경우가 있다.
  • Wireshark로 TLS Handshake 과정을 분석할 수 있다.

알아두면 좋은 내용

  • SSL보다 TLS 사용이 권장된다.
  • 인증서가 만료되면 HTTPS 경고가 발생할 수 있다.
  • HTTPS는 HTTP + TLS 조합이다.
  • TLS는 기밀성, 무결성, 인증 기능을 제공한다.

핵심 요약

  • TLS는 네트워크 통신을 암호화하는 보안 프로토콜이다.
  • HTTPS는 TLS를 이용해 안전한 웹 통신을 제공한다.
  • Handshake 과정에서 인증서 검증과 키 교환이 이루어진다.
  • 실제 데이터는 대칭키 방식으로 암호화된다.
  • 웹 보안과 인프라 보안을 이해하기 위한 핵심 기술이다.
