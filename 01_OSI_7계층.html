01. OSI 7계층

OSI(Open Systems Interconnection) 7계층은 네트워크 통신 과정을 역할별로 나누어 표현한 참조 모델이다. 실제 데이터가 반드시 7단계를 거치는 것은 아니지만, 어떤 프로토콜과 장비가 어떤 역할을 수행하는지 이해하기 위한 기준으로 사용된다. 네트워크를 공부할 때 가장 먼저 배우는 이유도 통신 과정을 단계별로 이해하기 쉽기 때문이다.

왜 중요한가?

  • 장애가 발생했을 때 어느 계층에서 문제가 생겼는지 빠르게 파악할 수 있다.
  • 방화벽, 스위치, 라우터가 어떤 계층에서 동작하는지 이해할 수 있다.
  • 패킷 분석과 로그 분석의 기본 개념이 된다.
  • 새로운 프로토콜을 공부할 때도 어느 계층에 속하는지 먼저 생각하면 이해가 쉬워진다.

핵심 이론

OSI 모델은 총 7개의 계층으로 구성된다. 상위 계층은 사용자와 가까운 기능을 담당하고, 하위 계층은 실제 데이터를 전송하는 역할을 담당한다.

계층역할대표 프로토콜 / 장비
7. Application사용자가 직접 이용하는 서비스 제공HTTP, HTTPS, DNS
6. Presentation암호화, 복호화, 인코딩TLS, SSL
5. Session세션 생성 및 유지NetBIOS
4. Transport신뢰성 있는 데이터 전송TCP, UDP
3. NetworkIP 주소를 이용한 라우팅IP, ICMP, Router
2. Data LinkMAC 주소 기반 통신Ethernet, Switch
1. Physical전기 신호 전달UTP, 광케이블

계층Protocol Data Unit
Application ~ SessionData
TransportSegment(TCP) / Datagram(UDP)
NetworkPacket
Data LinkFrame
PhysicalBit

데이터를 보낼 때는 상위 계층에서 하위 계층으로 내려가며 필요한 헤더가 추가된다. 이 과정을 캡슐화라고 한다. 반대로 데이터를 받을 때는 하위 계층부터 헤더를 제거하며 올라오는데 이를 역캡슐화라고 한다.

흐름도

Application Data ↓ TCP Header 추가 ↓ IP Header 추가 ↓ Ethernet Header 추가 ↓ Frame 전송

Frame 수신 ↓ Ethernet Header 제거 ↓ IP Header 제거 ↓ TCP Header 제거 ↓ Application Data 전달

예시

웹 브라우저에서 https://example.com 접속 DNS 조회 ↓ TCP 연결 ↓ TLS 연결 ↓ HTTP Request ↓ HTTP Response

실무에서 사용하는 경우

  • 웹 사이트 접속이 되지 않을 경우 DNS → TCP → TLS → HTTP 순서로 점검한다.
  • 스위치 문제는 2계층(MAC, VLAN), 라우팅 문제는 3계층(IP)을 우선 확인한다.
  • 방화벽 정책은 주로 IP와 Port(3~4계층)를 기준으로 작성한다.
  • Wireshark에서 패킷을 분석할 때도 어느 계층의 정보인지 구분하면서 확인한다.

알아두면 좋은 내용

  • OSI는 참조 모델이고, 실제 인터넷은 TCP/IP 모델을 기반으로 동작한다.
  • 스위치는 L2, 라우터는 L3 장비로 구분하는 경우가 많다.
  • HTTP는 7계층, TCP는 4계층, IP는 3계층 프로토콜이다.
  • 네트워크 문제를 분석할 때는 계층을 하나씩 내려가며 확인하면 원인을 찾기 쉽다.

핵심 요약

  • OSI 7계층은 네트워크를 기능별로 구분한 참조 모델이다.
  • 각 계층은 서로 다른 역할을 수행하며 데이터를 전달한다.
  • 송신은 캡슐화, 수신은 역캡슐화 과정으로 이루어진다.
  • 장애 분석, 패킷 분석, 보안 장비 이해의 기본이 되는 개념이다.
  • 모든 네트워크 개념을 공부할 때 기준이 되는 가장 중요한 이론이다.
