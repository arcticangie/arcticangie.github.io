06. Gateway & Routing

Gateway는 다른 네트워크로 나가기 위한 출입구 역할을 하는 장비(또는 주소)이다. Routing은 목적지 IP를 확인한 뒤 가장 적절한 경로를 선택해 패킷을 전달하는 과정이다. 같은 네트워크에서는 Gateway를 거치지 않지만, 다른 네트워크와 통신할 때는 반드시 Gateway를 통해 이동한다.

왜 중요한가?

  • 외부 네트워크와 통신하기 위해 반드시 필요한 개념이다.
  • 라우터가 패킷을 어디로 전달하는지 이해할 수 있다.
  • 인터넷 연결 문제를 분석할 때 가장 먼저 확인하는 항목 중 하나이다.
  • 방화벽과 라우터 정책을 이해하는 기본이 된다.

핵심 이론

용어설명
Gateway다른 네트워크로 나가는 기본 경로
Routing목적지까지의 최적 경로를 선택하는 과정
Routing Table목적지와 경로 정보를 저장하는 테이블
Next Hop다음으로 패킷을 전달할 장비
Default Route일치하는 경로가 없을 때 사용하는 기본 경로(0.0.0.0/0)

구분특징
Static Routing관리자가 직접 경로를 설정한다.
Dynamic Routing라우팅 프로토콜을 이용해 자동으로 경로를 학습한다.

라우터는 목적지 IP를 Routing Table과 비교한 뒤 가장 적합한 경로를 선택한다. 같은 Subnet이면 직접 통신하고, 다른 Subnet이면 Gateway를 통해 패킷을 전달한다.

흐름도

PC (192.168.10.10) ↓ 같은 Subnet 확인 ↓ Gateway (192.168.10.1) ↓ Router ↓ Internet

목적지 IP 확인 ↓ Routing Table 조회 ↓ Next Hop 결정 ↓ 패킷 전달

예시

PC : 192.168.0.10 Gateway : 192.168.0.1 ↓ 8.8.8.8 접속 ↓ Gateway로 패킷 전달 ↓ 라우터가 인터넷으로 전달

실무에서 사용하는 경우

  • 인터넷 연결이 안 될 때 Gateway 주소가 올바르게 설정되어 있는지 확인한다.
  • ipconfig, ip route, route print 명령으로 Gateway와 Routing Table을 확인한다.
  • Traceroute를 사용하면 패킷이 어떤 경로를 거쳐 이동하는지 확인할 수 있다.
  • 다른 대역과 통신이 되지 않는 경우 라우팅 설정이나 Default Route를 먼저 점검한다.

알아두면 좋은 내용

  • Gateway는 보통 같은 Subnet 안에 존재한다.
  • Default Gateway가 없으면 외부 네트워크와 통신할 수 없다.
  • Routing은 목적지 IP를 기준으로 이루어지며 MAC 주소는 L2에서 사용된다.
  • 기업 환경에서는 여러 개의 라우터가 Dynamic Routing으로 경로를 공유하는 경우가 많다.

핵심 요약

  • Gateway는 다른 네트워크로 나가기 위한 출입구 역할을 한다.
  • Routing은 목적지까지의 최적 경로를 선택하는 과정이다.
  • Routing Table에는 목적지와 전달 경로 정보가 저장된다.
  • 같은 네트워크는 직접 통신하고, 다른 네트워크는 Gateway를 거친다.
  • 네트워크 연결 문제를 분석할 때 반드시 이해해야 하는 핵심 개념이다.
