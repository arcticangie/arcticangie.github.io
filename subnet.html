04. Subnet (서브넷)

Subnet은 하나의 큰 네트워크를 여러 개의 작은 네트워크로 나누는 기술이다. 네트워크를 목적에 맞게 분리하면 브로드캐스트 범위를 줄일 수 있고, 관리와 보안 측면에서도 효율적이다. 기업 환경에서는 VLAN과 함께 거의 항상 사용된다.

왜 중요한가?

  • IP 주소를 효율적으로 사용할 수 있다.
  • 브로드캐스트 트래픽을 줄여 네트워크 성능을 높인다.
  • 부서별 또는 서비스별 네트워크를 분리할 수 있다.
  • 클라우드 VPC와 온프레미스 네트워크 설계의 기본 개념이다.

핵심 이론

용어설명
Subnet Mask네트워크 영역과 호스트 영역을 구분하는 값
CIDR/24, /16처럼 네트워크 비트 수를 표시하는 방식
Network Address네트워크 자체를 의미하는 첫 번째 주소
Broadcast Address동일 네트워크의 모든 장비에 전송하는 마지막 주소
Host Address실제 장비에 할당 가능한 주소

CIDRSubnet Mask사용 가능한 Host 수
/24255.255.255.0254
/25255.255.255.128126
/26255.255.255.19262
/27255.255.255.22430
/28255.255.255.24014

Host 개수는 2^(Host Bit) - 2로 계산한다. Network Address와 Broadcast Address는 장비에 할당하지 않기 때문에 2개를 제외한다.

흐름도

192.168.10.0/24 ↓ 192.168.10.0/25 (1~126) + 192.168.10.128/25 (129~254)

Client ↓ 동일 Subnet 확인 ↓ 같은 Subnet → Switch 다른 Subnet → Gateway

예시

192.168.1.0/24 Network Address 192.168.1.0 Host 192.168.1.1 ~ 192.168.1.254 Broadcast 192.168.1.255

실무에서 사용하는 경우

  • 부서별 네트워크를 분리할 때 서로 다른 Subnet을 할당한다.
  • AWS VPC나 Azure VNet을 구성할 때 Public Subnet과 Private Subnet을 나눈다.
  • 라우터는 목적지 IP와 Subnet Mask를 이용해 같은 네트워크인지 판단한다.
  • 네트워크 설계 시 필요한 Host 수를 계산해 적절한 CIDR을 선택한다.

알아두면 좋은 내용

  • /24는 가장 많이 사용하는 CIDR 중 하나이며 Host는 254개를 사용할 수 있다.
  • 같은 Subnet이면 Gateway를 거치지 않고 직접 통신한다.
  • 다른 Subnet과 통신할 때는 반드시 Gateway를 거친다.
  • Subnet과 VLAN은 함께 사용하는 경우가 많지만 서로 다른 개념이다.

핵심 요약

  • Subnet은 하나의 네트워크를 여러 개로 분리하는 기술이다.
  • Subnet Mask와 CIDR을 이용해 네트워크 범위를 결정한다.
  • Network Address와 Broadcast Address는 Host에 사용할 수 없다.
  • Host 개수는 2^(Host Bit)-2 공식으로 계산한다.
  • 기업 네트워크와 클라우드 환경 모두에서 반드시 이해해야 하는 개념이다.
