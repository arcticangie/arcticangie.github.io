12. Firewall

Firewall은 미리 정의한 보안 정책에 따라 네트워크 트래픽을 허용하거나 차단하는 보안 장비이다. 출발지와 목적지 IP, Port, Protocol 등을 확인하여 통신을 제어하며 기업 네트워크에서 가장 기본적인 보안 장비로 사용된다.

왜 중요한가?

  • 허가되지 않은 접근을 차단할 수 있다.
  • 내부 자산을 외부 공격으로부터 보호한다.
  • 서비스별 접근 정책을 세밀하게 관리할 수 있다.
  • 인프라 보안에서 가장 많이 다루는 장비 중 하나이다.

핵심 이론

용어설명
Policy허용 또는 차단 기준이 되는 규칙
ACL접근 제어 목록
Stateful세션 정보를 기억하여 통신을 검사
Stateless패킷 단위로 검사하며 세션을 저장하지 않음
Session허용된 통신 정보를 일정 시간 유지

구분StatefulStateless
세션 관리지원미지원
보안성높음상대적으로 낮음
현재 사용대부분의 방화벽일부 환경

방화벽은 일반적으로 출발지 IP, 목적지 IP, Port, Protocol을 기준으로 정책을 적용한다. 기업 환경에서는 Stateful Firewall을 가장 많이 사용한다.

흐름도

Client ↓ Firewall ↓ Policy 확인 ↓ 허용 ↓ Web Server

패킷 수신 ↓ IP / Port 확인 ↓ Session 확인 ↓ 허용 또는 차단

예시

Allow Source : Any Destination : Web Server TCP 443 ↓ HTTPS 접속 허용

실무에서 사용하는 경우

  • 서비스 오픈 시 필요한 Port만 허용하는 정책을 추가한다.
  • 접속이 되지 않으면 방화벽 로그와 정책을 먼저 확인한다.
  • 특정 IP를 차단하거나 허용하는 정책을 관리한다.
  • Session 정보를 확인하여 비정상적인 연결 여부를 분석한다.

알아두면 좋은 내용

  • 기본 정책은 필요한 통신만 허용하고 나머지는 차단하는 방식이 권장된다.
  • 방화벽은 NAT 기능을 함께 제공하는 경우가 많다.
  • L3/L4 정보뿐 아니라 L7(Application)까지 검사하는 차세대 방화벽(NGFW)도 많이 사용된다.
  • 허용 정책보다 정책의 순서가 결과에 영향을 주는 경우가 있다.

핵심 요약

  • Firewall은 네트워크 트래픽을 허용하거나 차단하는 보안 장비이다.
  • IP, Port, Protocol을 기준으로 정책을 적용한다.
  • Stateful Firewall이 가장 일반적으로 사용된다.
  • 기업 네트워크 보안의 핵심 장비이다.
  • 방화벽 정책과 로그를 읽을 수 있어야 인프라 보안 업무를 수행하기 쉽다.
