07. NAT (Network Address Translation)

NAT는 사설 IP와 공인 IP를 서로 변환해 주는 기술이다. 회사나 집에서는 대부분 사설 IP를 사용하지만 인터넷과 통신하려면 공인 IP가 필요하기 때문에, 라우터나 방화벽이 NAT를 수행해 외부와 통신할 수 있도록 한다. IPv4 주소 부족 문제를 해결하기 위해 가장 널리 사용하는 기술 중 하나이다.

왜 중요한가?

  • 사설 IP를 사용하는 내부 장비도 인터넷에 접속할 수 있다.
  • 공인 IP를 효율적으로 사용할 수 있다.
  • 기업 방화벽과 라우터에서 거의 항상 사용된다.
  • 방화벽 로그를 분석할 때 변환 전후 IP를 함께 확인해야 한다.

핵심 이론

종류설명
Static NAT사설 IP와 공인 IP를 1:1로 고정 매핑한다.
Dynamic NAT공인 IP 풀에서 사용 가능한 주소를 동적으로 할당한다.
PAT(NAPT)여러 사설 IP가 하나의 공인 IP를 포트 번호와 함께 공유한다.

항목내용
Inside Local내부에서 사용하는 사설 IP
Inside GlobalNAT 후 외부에 보이는 공인 IP
변환 위치라우터, 방화벽, NAT Gateway
주요 목적IPv4 주소 절약 및 내부망 보호

가정이나 회사에서 가장 많이 사용하는 방식은 PAT이다. 여러 대의 PC가 하나의 공인 IP를 공유하면서도 포트 번호를 이용해 각각의 세션을 구분한다.

흐름도

PC 1 (192.168.0.10) ↓ NAT ↓ 203.0.113.10:50001 ↓ Internet

PC 2 (192.168.0.20) ↓ NAT ↓ 203.0.113.10:50002 ↓ Internet

예시

사설 IP 192.168.10.15 ↓ NAT 수행 ↓ 공인 IP 203.0.113.100 ↓ 웹 서버 접속

실무에서 사용하는 경우

  • 인터넷은 되지만 특정 서비스만 접속되지 않는 경우 NAT 정책을 확인한다.
  • 방화벽 로그에서 변환 전 IP와 변환 후 IP를 함께 확인한다.
  • AWS NAT Gateway를 이용해 Private Subnet의 서버가 인터넷으로 업데이트를 받도록 구성한다.
  • 내부 서버를 외부에 공개할 때는 Static NAT 또는 Port Forwarding을 사용한다.

알아두면 좋은 내용

  • NAT는 보안 기능이 아니라 주소 변환 기술이다.
  • PAT는 NAT의 가장 일반적인 형태이며 하나의 공인 IP를 여러 장비가 공유한다.
  • IPv6 환경에서는 주소 공간이 넓어 NAT 의존도가 낮아진다.
  • 클라우드 환경에서도 NAT Gateway는 자주 사용하는 서비스이다.

핵심 요약

  • NAT는 사설 IP와 공인 IP를 변환하는 기술이다.
  • IPv4 주소 부족 문제를 해결하기 위해 널리 사용된다.
  • Static NAT, Dynamic NAT, PAT 세 가지 방식을 알아두면 된다.
  • 기업 환경에서는 대부분 PAT를 사용한다.
  • 라우터와 방화벽을 이해하려면 NAT 개념은 반드시 알아야 한다.
