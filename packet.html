16. Packet

Packet은 네트워크를 통해 전달되는 데이터 단위이다. 사용자가 전송하는 데이터는 여러 개의 Packet으로 분할되어 전송되며, 각 Packet에는 목적지까지 전달하기 위한 다양한 Header 정보가 포함된다. 패킷 구조를 이해하면 네트워크 흐름과 보안 로그를 분석하기 쉬워진다.

왜 중요한가?

  • 네트워크 통신이 이루어지는 기본 단위를 이해할 수 있다.
  • Wireshark와 같은 패킷 분석 도구를 사용할 때 필수 개념이다.
  • 네트워크 장애와 보안 이벤트 분석의 기본이 된다.
  • Header 정보를 통해 통신 과정을 추적할 수 있다.

핵심 이론

구성 요소설명
Ethernet Header출발지/목적지 MAC 주소 정보
IP Header출발지/목적지 IP 주소 정보
TCP/UDP HeaderPort, Sequence 등 전송 정보
Payload실제 전송되는 데이터

계층데이터 단위(PDU)
TransportSegment(TCP) / Datagram(UDP)
NetworkPacket
Data LinkFrame
PhysicalBit

송신 측에서는 Header를 추가하며 데이터를 전송하고, 수신 측에서는 Header를 제거하면서 원래 데이터를 복원한다.

흐름도

Application Data ↓ TCP Header ↓ IP Header ↓ Ethernet Header ↓ Packet 전송

Packet 수신 ↓ Ethernet Header 제거 ↓ IP Header 제거 ↓ TCP Header 제거 ↓ Application Data

예시

브라우저에서 웹 페이지 요청 ↓ HTTP 데이터 생성 ↓ Packet 생성 ↓ 네트워크 전송 ↓ 서버 응답

실무에서 사용하는 경우

  • Wireshark에서 Header 정보를 확인하여 통신 과정을 분석한다.
  • IP 주소와 Port를 확인해 통신 상대를 식별한다.
  • Packet 손실이나 재전송 여부를 확인해 장애 원인을 분석한다.
  • 보안 장비 로그와 Packet 정보를 함께 비교하여 이상 통신을 확인한다.

알아두면 좋은 내용

  • Packet에는 Header와 Payload가 포함된다.
  • Header에는 통신에 필요한 다양한 정보가 저장된다.
  • Packet 분석은 침해사고 대응에서도 자주 사용된다.
  • 캡슐화와 역캡슐화를 함께 이해하면 Packet 구조를 이해하기 쉽다.

핵심 요약

  • Packet은 네트워크를 통해 전달되는 데이터 단위이다.
  • Ethernet, IP, TCP/UDP Header와 Payload로 구성된다.
  • 패킷 구조를 이해하면 네트워크 분석이 쉬워진다.
  • Wireshark와 보안 로그 분석의 핵심 개념이다.
  • 인프라 보안을 공부할 때 반드시 익혀야 하는 기본 내용이다.
