13. VPN (Virtual Private Network)

VPN은 인터넷과 같은 공용 네트워크 위에서 암호화된 통신 터널을 만들어 안전하게 데이터를 주고받는 기술이다. 외부에서도 회사 내부 네트워크에 안전하게 접속할 수 있기 때문에 재택근무, 지사 연결, 원격 관리 환경에서 많이 사용된다.

왜 중요한가?

  • 공용 네트워크에서도 안전하게 통신할 수 있다.
  • 원격 근무 환경에서 내부 시스템에 접근할 수 있다.
  • 데이터를 암호화하여 도청 위험을 줄인다.
  • 기업 인프라에서 자주 사용하는 보안 기술이다.

핵심 이론

종류설명
Site-to-Site VPN본사와 지사처럼 네트워크와 네트워크를 연결한다.
Remote Access VPN사용자가 외부에서 회사 내부망에 접속한다.
IPSec VPNIP 계층에서 암호화하여 안전하게 통신한다.
SSL VPN웹 브라우저 또는 전용 클라이언트를 이용해 접속한다.

구분IPSec VPNSSL VPN
동작 계층Network LayerApplication Layer
주요 용도지사 간 연결원격 사용자 접속
특징안정적인 터널 연결사용이 간편

VPN은 데이터를 암호화한 뒤 전송하기 때문에 중간에서 패킷을 가로채더라도 내용을 확인하기 어렵다.

흐름도

사용자 ↓ VPN Client ↓ 암호화 터널 생성 ↓ VPN Gateway ↓ 사내 네트워크

지사 Router ↓ IPSec VPN ↓ 본사 Firewall ↓ 내부 서버

예시

재택근무 ↓ SSL VPN 접속 ↓ 사내 ERP 서버 접속 ↓ 암호화된 통신 유지

실무에서 사용하는 경우

  • 재택근무 사용자의 내부 시스템 접속을 지원한다.
  • 본사와 지사를 인터넷을 통해 안전하게 연결한다.
  • VPN 연결 실패 시 인증 정보, 터널 상태, 방화벽 정책을 확인한다.
  • 관리자는 VPN 접속 로그를 확인하여 비정상 접속 여부를 점검한다.

알아두면 좋은 내용

  • VPN은 암호화 통신을 제공하지만 사용자 인증도 함께 중요하다.
  • 다중 인증(MFA)을 함께 사용하는 경우가 많다.
  • SSL VPN은 원격 사용자, IPSec VPN은 네트워크 간 연결에 많이 사용된다.
  • VPN Gateway는 방화벽 장비에서 함께 제공되는 경우도 많다.

핵심 요약

  • VPN은 공용 네트워크에서 안전한 통신을 제공하는 기술이다.
  • 대표적으로 IPSec VPN과 SSL VPN이 있다.
  • 원격 근무와 지사 연결 환경에서 많이 사용된다.
  • 데이터를 암호화하여 기밀성을 높인다.
  • 기업 인프라 보안에서 자주 사용하는 핵심 기술이다.
