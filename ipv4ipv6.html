03. IPv4 / IPv6

IP 주소는 네트워크에 연결된 장비를 식별하기 위한 주소이다. 인터넷에서 데이터를 주고받기 위해서는 출발지와 목적지의 IP 주소가 필요하며, 현재는 IPv4와 IPv6 두 가지 주소 체계를 함께 사용하고 있다. 실무에서는 아직 IPv4가 가장 많이 사용되지만 IPv6 환경도 점점 늘어나고 있다.

왜 중요한가?

  • 네트워크 통신의 출발지와 목적지를 식별하는 가장 기본적인 정보이다.
  • 방화벽 정책, 라우팅, NAT 등 대부분의 네트워크 설정에 사용된다.
  • 로그 분석 시 공격자 또는 대상 시스템을 식별하는 기준이 된다.
  • 사설 IP와 공인 IP를 구분할 수 있어야 네트워크 구조를 이해하기 쉽다.

핵심 이론

표현 방식192.168.0.12001:db8::1

현재 사용가장 많이 사용점차 확대 중

종류주소 범위설명

IPv4는 주소 부족 문제로 NAT를 많이 사용한다. IPv6는 훨씬 많은 주소를 제공하기 때문에 NAT 의존도를 줄일 수 있으며, 자동 주소 설정 기능도 지원한다.

흐름도

PC (192.168.0.10) ↓ Gateway ↓ NAT ↓ Public IP ↓ Internet ↓ Web Server

PC 실행 ↓ DHCP 요청 ↓ IP 주소 할당 ↓ Gateway 설정 ↓ 인터넷 통신 가능

예시

회사 PC : 192.168.10.15 Gateway : 192.168.10.1 ↓ 외부 사이트 접속 ↓ 공인 IP로 변환(NAT) ↓ 인터넷 통신

실무에서 사용하는 경우

  • 방화벽 정책 작성 시 출발지와 목적지 IP를 지정한다.
  • 로그에서 공격자의 IP 주소를 확인해 차단하거나 추적한다.
  • 사설 IP는 내부 통신, 공인 IP는 외부 통신에 사용되는지 확인한다.
  • 169.254.x.x 주소가 보이면 DHCP 문제를 먼저 의심한다.

알아두면 좋은 내용

  • IPv4는 주소 부족 문제 때문에 NAT가 거의 필수적으로 사용된다.
  • 127.0.0.1은 localhost로 자기 자신을 의미한다.
  • IPv6는 주소 공간이 매우 넓어 IoT 환경에서도 많이 사용된다.
  • IP 주소만으로는 통신 상대를 찾을 수 없으며 MAC 주소와 함께 사용된다.

핵심 요약

  • IP 주소는 네트워크 장비를 식별하기 위한 주소이다.
  • IPv4는 32bit, IPv6는 128bit 주소 체계를 사용한다.
  • Private IP는 내부망, Public IP는 인터넷에서 사용한다.
  • Loopback과 APIPA 주소는 자주 등장하므로 함께 기억한다.
  • IP 주소는 라우팅과 방화벽 정책의 가장 기본이 되는 정보이다.
