14. HTTP & HTTPS

HTTP는 웹 브라우저와 웹 서버가 데이터를 주고받기 위한 프로토콜이다. HTTPS는 HTTP에 TLS 암호화를 추가한 방식으로, 로그인 정보나 개인정보처럼 중요한 데이터를 안전하게 전송할 수 있다. 현재 대부분의 웹 서비스는 HTTPS를 기본으로 사용한다.

왜 중요한가?

  • 웹 서비스의 기본 통신 방식을 이해할 수 있다.
  • 웹 보안과 패킷 분석의 기초가 된다.
  • HTTP 요청과 응답 구조를 이해하면 로그 분석이 쉬워진다.
  • 기업 서비스는 대부분 HTTPS를 사용한다.

핵심 이론

구분HTTPHTTPS
기본 Port80443
암호화미지원TLS 사용
보안성낮음높음
사용 환경일반 웹로그인, 결제 등

Method설명
GET데이터 조회
POST데이터 생성 및 전송
PUT데이터 수정
DELETE데이터 삭제

HTTP 통신은 Request와 Response 구조로 이루어진다. 요청에는 Method, Header, Body가 포함될 수 있으며 서버는 Status Code와 데이터를 응답한다.

흐름도

Browser ↓ HTTPS 요청 ↓ TLS 연결 ↓ Web Server ↓ Response

Client ↓ HTTP Request ↓ Server ↓ HTTP Response

예시

GET /index.html ↓ Status Code : 200 OK ↓ 웹 페이지 표시

실무에서 사용하는 경우

  • 웹 서버 장애 시 HTTP Status Code를 확인한다.
  • 브라우저 개발자 도구와 Wireshark로 Request/Response를 분석한다.
  • HTTPS 인증서 오류가 발생하면 인증서와 TLS 설정을 점검한다.
  • 방화벽에서 TCP 80, TCP 443 허용 여부를 확인한다.

알아두면 좋은 내용

  • HTTPS는 HTTP보다 안전한 통신을 제공한다.
  • 대표적인 Status Code는 200, 301, 403, 404, 500이다.
  • Cookie와 Session은 사용자 상태를 유지하기 위해 사용된다.
  • HTTPS는 TLS 위에서 동작한다.

핵심 요약

  • HTTP는 웹 통신 프로토콜이다.
  • HTTPS는 TLS를 이용해 데이터를 암호화한다.
  • GET, POST는 가장 많이 사용하는 HTTP Method이다.
  • 웹 분석에서는 Request, Response, Status Code를 자주 확인한다.
  • 현대 웹 서비스는 대부분 HTTPS를 기본으로 사용한다.
