17. Wireshark

Wireshark는 네트워크를 통해 오가는 Packet을 캡처하고 분석하는 대표적인 네트워크 분석 도구이다. Packet의 Header와 Payload를 확인할 수 있어 네트워크 장애 분석과 보안 사고 대응에서 가장 많이 사용하는 프로그램 중 하나이다.

왜 중요한가?

  • 실제 네트워크 통신 과정을 눈으로 확인할 수 있다.
  • TCP, DNS, HTTP, TLS 등 다양한 프로토콜을 분석할 수 있다.
  • 통신 장애 원인과 비정상 트래픽을 확인하는 데 도움이 된다.
  • 보안 관제와 침해사고 대응에서도 자주 사용된다.

핵심 이론

기능설명
Packet Capture실시간 네트워크 트래픽 수집
Protocol Analysis프로토콜별 Header 분석
Display Filter원하는 Packet만 조회
Follow Stream하나의 통신 흐름 확인

자주 사용하는 Filter설명
ip.addr == 192.168.0.10특정 IP 조회
tcp.port == 443HTTPS 통신 조회
dnsDNS Packet 조회
httpHTTP Packet 조회
icmpPing 통신 조회

Packet을 시간 순서대로 확인하면 통신이 정상적으로 이루어졌는지 쉽게 파악할 수 있다.

흐름도

Packet Capture ↓ Filter 적용 ↓ Header 확인 ↓ 통신 흐름 분석

SYN ↓ SYN/ACK ↓ ACK ↓ HTTP Request ↓ HTTP Response

예시

dns ↓ example.com 조회 ↓ A Record 응답 ↓ TCP 연결 시작

실무에서 사용하는 경우

  • 웹 접속이 실패하면 TCP Handshake가 정상인지 확인한다.
  • DNS 조회가 실패했는지 Packet으로 확인한다.
  • 재전송(Retransmission), Reset(RST), Time Out 여부를 분석한다.
  • 의심스러운 외부 IP와의 통신이 있는지 Packet을 확인한다.

알아두면 좋은 내용

  • Display Filter와 Capture Filter는 서로 다르다.
  • Packet 번호와 시간을 함께 보면 흐름을 이해하기 쉽다.
  • TCP Stream 기능으로 하나의 연결만 추적할 수 있다.
  • Packet 분석은 네트워크와 보안 분야 모두에서 자주 사용된다.

핵심 요약

  • Wireshark는 대표적인 Packet 분석 도구이다.
  • TCP, DNS, HTTP, TLS 등 다양한 프로토콜을 분석할 수 있다.
  • Filter를 사용하면 원하는 Packet만 빠르게 찾을 수 있다.
  • 장애 분석과 보안 사고 대응에서 매우 자주 사용된다.
  • 네트워크 공부를 할 때 반드시 익혀두는 것이 좋다.
