05. DNS (Domain Name System)

DNS는 사람이 기억하기 쉬운 도메인 이름을 IP 주소로 변환하는 서비스이다. 브라우저에서 사이트에 접속할 때 가장 먼저 수행되는 과정 중 하나이며, DNS가 정상적으로 동작하지 않으면 IP 주소를 직접 입력하지 않는 이상 원하는 사이트에 접속할 수 없다.

왜 중요한가?

  • 도메인과 IP 주소를 연결하는 핵심 서비스이다.
  • 웹 서비스, 메일 서비스 등 대부분의 인터넷 서비스가 DNS를 사용한다.
  • 보안 분석 시 악성 도메인 조회, DNS 로그 분석에 자주 활용된다.
  • 네트워크 장애 발생 시 가장 먼저 확인하는 항목 중 하나이다.

핵심 이론

용어설명
Domainexample.com과 같은 사람이 읽기 쉬운 주소
DNS Server도메인과 IP 정보를 관리하는 서버
Recursive Query최종 IP를 찾을 때까지 대신 조회
Iterative Query다음 DNS 서버 정보를 순차적으로 안내
Cache조회 결과를 일정 시간 저장하여 속도 향상

레코드용도
A도메인 → IPv4 주소
AAAA도메인 → IPv6 주소
CNAME별칭(Alias) 지정
MX메일 서버 정보
TXT인증 및 기타 텍스트 정보 저장
PTRIP → 도메인 역방향 조회

DNS는 기본적으로 UDP 53번 포트를 사용하며, 영역 전송이나 큰 응답은 TCP 53번을 사용할 수도 있다.

흐름도

사용자 ↓ 브라우저 ↓ DNS Cache 확인 ↓ DNS Server 질의 ↓ IP 주소 응답 ↓ 웹 서버 접속

example.com ↓ A Record 조회 ↓ 203.0.113.10 ↓ TCP 연결 시작

예시

사용자 입력 https://example.com ↓ DNS 서버 조회 ↓ 93.184.216.xxx ↓ 브라우저가 해당 IP로 접속

실무에서 사용하는 경우

  • 웹 사이트 접속이 되지 않으면 먼저 DNS 조회가 정상인지 확인한다.
  • nslookup, dig 명령으로 도메인 조회 결과를 확인한다.
  • 악성코드가 수상한 도메인과 통신하는지 DNS 로그를 분석한다.
  • 피싱 사이트는 정상 사이트와 비슷한 도메인을 사용하는 경우가 많아 도메인 확인이 중요하다.

알아두면 좋은 내용

  • DNS Cache 덕분에 같은 도메인은 반복 조회 속도가 빠르다.
  • hosts 파일에 등록된 정보가 DNS보다 우선 적용된다.
  • DNS가 정상이어도 방화벽이나 라우팅 문제로 접속이 실패할 수 있다.
  • DNS는 단순 이름 변환 서비스지만 보안 측면에서도 매우 중요한 역할을 한다.

핵심 요약

  • DNS는 도메인을 IP 주소로 변환하는 서비스이다.
  • 기본적으로 UDP 53번 포트를 사용한다.
  • A, AAAA, CNAME, MX, TXT, PTR 레코드는 반드시 알아두는 것이 좋다.
  • 웹 접속 문제와 보안 분석에서 가장 자주 확인하는 서비스 중 하나이다.
  • DNS 동작 과정을 이해하면 네트워크 흐름도 함께 이해하기 쉬워진다.
